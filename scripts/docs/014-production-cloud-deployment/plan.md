# Implementation Plan: Production Cloud Deployment (Render)

This plan outlines the specific steps required to deploy the `GlutenFreeAPI` backend and its associated PostgreSQL database to Render, and update the client-side configuration.

## 🛠️ Step-by-Step Implementation

### Step 1: Declare Infrastructure as Code (`render.yaml`)
1. Create a `render.yaml` file at the root of the repository.
2. Define a database resource named `freeglu-db` using PostgreSQL and a free plan.
3. Define a web service resource named `freeglu-api` using the `Docker` environment, `GlutenFreeAPI` as the root directory, and link the `DATABASE_URL` directly from the database resource. Add an autogenerated `JWT_SECRET`.

### Step 2: Implement Client-Side Environment Toggle
1. Add `USE_LOCAL_BACKEND` (Boolean) and `CLOUD_BACKEND_URL` (String) inside `Platform.kt` in `commonMain` directory.
2. Update all platform implementations of `getApiBaseUrl()` (`Platform.android.kt`, `Platform.ios.kt`, `Platform.js.kt`, `Platform.wasmJs.kt`) to resolve to the `CLOUD_BACKEND_URL` if `USE_LOCAL_BACKEND = false`.

### Step 3: Configure CI/CD Deploy Trigger
1. Update `.github/workflows/ci.yml`.
2. Add a `Trigger Render Deploy` step in the `backend-tests` job.
3. Protect the step to run only when pushing to the `main` branch, succeeding if the tests pass, and executing `curl -f -X POST` to the Render deploy hook URL if the `RENDER_DEPLOY_HOOK_URL` repository secret is present.
