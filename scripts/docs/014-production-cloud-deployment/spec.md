# Feature Specification: Production Cloud Deployment (Render)

**Feature Branch**: `014-production-cloud-deployment`
**Status**: Draft (Under Review)

## Vision & Product Overview
To transition FreeGluApp from a local-only prototype into a live, globally accessible application, we need to host the `GlutenFreeAPI` (Swift Vapor backend) and its associated PostgreSQL database in a production cloud environment. By deploying to **Render**, we ensure that client apps (Android, iOS, JS, Wasm) can retrieve, search, and manage products and user favorites from any location, without requiring a local database or local server instance.

---

## Technical Scope

### 1. Cloud Infrastructure (Render Blueprints)
- **Infrastructure as Code (IaC)**: Provide a `render.yaml` blueprint declaring:
  - A managed PostgreSQL instance (`freeglu-db`) on the free tier.
  - A Docker-based Web Service (`freeglu-api`) pointing to the `GlutenFreeAPI` root directory.
- **Environment Variables**:
  - `DATABASE_URL`: Injected dynamically from the PostgreSQL resource.
  - `JWT_SECRET`: Autogenerated as a secure, high-entropy random string.
  - `PORT`: Set to `8080` (or dynamically mapped by Render).

### 2. Client Platform Adaptability (`FreeGluKMP`)
- **Environment Toggles**: Add `USE_LOCAL_BACKEND` and `CLOUD_BACKEND_URL` in `Platform.kt` to allow seamless toggling between local dev and live server.
- **Platform Implementations**: Align `getApiBaseUrl()` in Android, iOS, JS, and WasmJs to respect the toggle, falling back to local emulators/localhost only when `USE_LOCAL_BACKEND = true`.

### 3. Continuous Integration & Deployment (GitHub Actions)
- **Automatic Deploys**: Enhance `.github/workflows/ci.yml` with a safe, test-gated step to trigger a fresh deploy to Render via a secure deploy hook when pushing to `main`.

---

## Criterios de Aceptación
- [ ] El archivo `render.yaml` declara de forma correcta la base de datos PostgreSQL y el servicio web Vapor utilizando la receta de Blueprint de Render.
- [ ] Todas las plataformas del cliente KMP (`android`, `iosSimulatorArm64`, `js`, `wasmJs`) compilan de manera exitosa y respetan el toggle de URL.
- [ ] La pipeline de GitHub Actions compila el backend, pasa los tests e intenta disparar el deploy hook de manera segura (con advertencias si el secret no está configurado).
